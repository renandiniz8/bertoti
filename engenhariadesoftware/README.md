Atividade 1: 
A principal diferença entre Engenharia de Software para as demais é a entrega de algo intangível. Enquanto nos outros ramos de Engenharia há uma entrega palpável, na engenharia de software não entrega nada físico.

Atividade 2: 
Engenharia de Software se difere da progamação devido ao foco em estabelecer parâmetros de estruturação do código, buscando melhorar a usabilidade, avaliar a inserção de novas aplicações de maneira otimizada e deixar sempre o software apto a receber novas atualizações sem prejudicar sua espinhal dorsal.

Atividade 3: 
Segurança: O software deve garantir a segurança dos usuários para que as operações e dados realizados sejam concluídos e armazenados de uma maneira que não comprometa e não traga prejuízo para nenhuma parte envolvida.
Integridade dos dados: As informações inseridas no software devem ser consistentes, precisas e rastreáveis. Garantindo que aquilo inserido não sofra alteração sem prévio conhecimento e autorização das partes envolvidas. 
Escalabilidade: Os engenheiros de software devem se preocupar com a quantidade de acessos simultâneos das pessoas de forma que possa garantir o perfeito funcionamento das ferramentas propostas. Por mais que inicialmente não tenha uma alta demanda, deve haver uma previsão de como este crescimento deve ser estruturado e quais os resultados esperados com essa previsão.
Extensão: Dado o avanço significativo das tecnologias, há diversos dispositivos distintos que podem acessar o software desenvolvido. A engenharia de software tem como objetivo o correto funcionamento das ferramentas independe do dispositivo que está sendo usado para acessar, sem comprometer a interação com o usuário e a troca das informações.
Agilidade: Os bons desenvolvedores atentam-se para que o software não rode etapas inúteis para alcançar os objetivos dos usuários. Procuram otimizar os passos e assim otimizar memória e velocidade de desempenho.

Citar e descrever 3 cenários de trade-offs (ver slide 12, mas usar outros exemplos)
Segurança x Facilidade: aplicativos de transporte garantem uma facilidade para o usuário mas apresenta riscos aos usários devido a limitação de critérios dos usuários.
Consistência x Disponibilidade: alguns bancos de dados priorizam a consistência (como a utilizada por bancos) enquanto outras tem como prioridade a garantia da disponibilidade (redes sociais).
Generalização x Características Específicas: Existem softwares orientados para atividades de um ramo, porém cada empresa do ramo possui um procedimento interno distinto.
